# Google Apps Script POS 系統 (點餐收銀機)

這是一個極度輕量、專為 iPad 操作最佳化的 Web POS (端點銷售系統)。採用純前端語法 (Vanilla JS / CSS) 搭配 Google Apps Script (GAS) 開發，並以 Google Sheets 作為免費的後端資料庫。

## ✨ 核心特色

- **iPad 完美相容**：採用 iOS PWA 模式 (加入主畫面即可全螢幕執行，無 Safari 工具列)，排版完美適配 iPad。
- **免架設後端**：完全依賴 Google Apps Script 與 Google Sheets，無須負擔伺服器與資料庫費用。
- **直覺的操作介面**：
  - 左側分類商品點選，支援多分類動態切換。
  - 右側購物車即時計算金額，一鍵清空或結帳。
- **強大的結帳模組**：
  - **自訂數字鍵盤**：內建結帳數字鍵與快速金額按鈕 (`+100`, `剛好`)，自動計算找零。
  - **一鍵快速折價**：支援結帳時快速套用折扣 (9折、8折、折10元、折50元)，折抵明細將完整同步至資料庫。
- **銷售軌跡與防弊**：
  - 內建「銷售紀錄」查詢面板。
  - 支援訂單「作廢 (Void)」，作廢後保留稽核軌跡，不會從試算表中刪除紀錄，有效防弊。
- **本機開發模式**：內建 Mock Data 機制，即便沒有在 GAS 環境下，也可以用瀏覽器直接開啟 `.html` 測試前端介面與功能。

## 📂 專案結構

本專案主要由 4 個 GAS 檔案組成：

- `Code.js` (或 `Code.gs`)：GAS 後端伺服器程式碼。負責處理 API 請求、寫入試算表資料。
- `Index.html`：網頁的主結構與進入點。
- `style.html`：所有的 CSS 樣式，大量使用 CSS Variables 管理主題色彩與 RWD 設計。
- `script.html`：前端的 JavaScript 邏輯，負責狀態管理、購物車操作與結帳流程。

## 🚀 部署教學 (Deployment)

1. **準備 Google Sheets 資料庫**：
   - 建立一個新的 Google 試算表。
   - 建立一個名為 `商品清單` 的工作表，第一行標題設為：`分類 | 品名 | 單價`。(可自由往下新增商品)
   - 建立一個名為 `訂單紀錄` 的工作表，第一行標題設為：`訂單時間 | 處理人員 | 總金額 | 購買明細 | 狀態`。
   
2. **複製程式碼**：
   - 在試算表點擊上方選單 `擴充功能` > `Apps Script`。
   - 建立 4 個檔案並分別將本專案的 `Code.js`, `Index.html`, `style.html`, `script.html` 內容貼入。

3. **發布 Web App**：
   - 點擊右上角 `部署` > `新增部署作業`。
   - 類型選擇 `網頁應用程式`。
   - 執行身分：`我`。
   - 誰可以存取：`所有人`。
   - 點擊 `部署` 後，即可獲得一個 `/exec` 結尾的公開網址。

4. **加到 iPad 主畫面**：
   - 用 iPad 的 Safari 開啟您的 `/exec` 網址。
   - 點擊 Safari 的分享按鈕 > `加入主畫面`。
   - 從桌面圖示開啟，即可享受全螢幕的收銀體驗！

## 🛠 開發與除錯

- **更新部署**：當您修改了 `Code.js` 的後端邏輯，請務必點擊 `部署` > `管理部署作業` > `編輯` > 選擇 `建立新版本` 進行更新。
- **測試網址**：平常開發前端時，可點擊 `部署` > `測試部署作業` 取得 `/dev` 結尾的網址，每次重新整理都會反映最新程式碼，不需重複發布。
- **錯誤捕捉**：腳本內建 `window.onerror`，只要遇到 JavaScript 崩潰皆會跳出 Alert 提示錯誤行號，方便在封閉的 GAS 環境中除錯。
